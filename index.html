SQL Injection

SQL Injection is a code injection technique that exploits vulnerabilities in the database layer of an application. Attackers can insert malicious SQL statements that can read, modify, or delete data from the database.

Severity: High - SQL Injection can lead to data theft, data loss, or even complete system compromise.

How It Works

SQL Injection occurs when user input is incorrectly filtered and directly incorporated into SQL queries:

1
2
3

Example of Vulnerable Code

// Vulnerable PHP code
$username = $_POST['username'];
$query = "SELECT * FROM users WHERE username = '$username'";
$result = $conn->query($query);

Prevention Techniques

To prevent SQL Injection attacks, always:

  • Use parameterized queries or prepared statements
  • Apply input validation and sanitization
  • Implement least privilege principle for database accounts
  • Use ORM (Object-Relational Mapping) libraries

Secure Code Example

// Secure PHP code using prepared statements
$stmt = $conn->prepare("SELECT * FROM users WHERE username = ?");
$stmt->bind_param("s", $_POST['username']);
$stmt->execute();
$result = $stmt->get_result();

Cross-Site Scripting (XSS)

Cross-Site Scripting (XSS) is a vulnerability that allows attackers to inject malicious client-side scripts into web pages viewed by other users. This can lead to session hijacking, credential theft, and more.

Severity: High - XSS can lead to account compromise, data theft, and website defacement.

Types of XSS

Reflected XSS
Stored XSS
DOM-based XSS

Reflected XSS

Occurs when malicious script is reflected off a web server, such as in search results or error messages. The attack is delivered through URLs.

Stored XSS

The malicious script is stored on the target server, such as in a database, and is later retrieved and displayed to users who access the affected page.

DOM-based XSS

Occurs entirely in the browser when JavaScript modifies the DOM environment in an unsafe way. The server is not involved in this type of attack.
