Understanding Cross-Site Request Forgery (CSRF)

Protect your web applications from unauthorized actions through forged requests

What is Cross-Site Request Forgery?

Cross-Site Request Forgery (CSRF) is a web security vulnerability that allows attackers to trick users into performing unwanted actions on websites where they're already authenticated. The attack exploits the trust that a website has in a user's browser.

Critical Security Risk

Common CSRF Attack Targets

  • Financial transactions
  • Account settings changes
  • Email/Password updates
  • E-commerce actions

How CSRF Attacks Work

1

User Authentication

User logs into a legitimate website and receives authentication cookies

2

Malicious Link

Attacker tricks user into clicking a link or visiting a malicious site

3

Forged Request

Malicious site sends unauthorized requests to the legitimate site using user's cookies

4

Action Execution

Legitimate site processes the request thinking it came from the authenticated user

CSRF Prevention Techniques

CSRF Token Implementation


// Generate CSRF Token
const csrfToken = crypto.randomBytes(32).toString('hex');

// Add to Form
<form action="/transfer" method="POST">
    <input type="hidden" name="_csrf" value="${csrfToken}">
    ...
</form>

Security Best Practices

  • Use CSRF Tokens
  • Same-Site Cookie Attribute
  • Custom Request Headers
  • Double Submit Cookie
